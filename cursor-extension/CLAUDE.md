# Algorithm Visualizer — Cursor Extension

## What this project is

A [Cursor](https://cursor.com)-first extension that lets developers select Python code, trigger a command, and get an AI-generated interactive step-by-step visualizer in a docked side panel. As the user steps through frames, the corresponding source line is highlighted in the editor in real time (bidirectional sync).

Visualizations are generated by calling the Claude API with a detailed prompt that instructs it to produce a complete self-contained HTML file.

## Relationship to sibling builds

| Directory | Target | Status |
|-----------|--------|--------|
| `cursor-extension/` (this) | Cursor | **Active** |
| `vscode-extension/` | VS Code (vanilla) | Active, source-of-truth for shared logic |
| `algo-visualizer-intellij/` | IntelliJ Platform | Paused |

The Cursor build is a fork of `vscode-extension/` with Cursor-specific branding, publish flow (Open VSX), and a runtime `isCursor()` check. Core logic (prompt, webview bridge, editor sync) should stay in lockstep with `vscode-extension/`.

## Architecture

```
src/
├── extension.ts        Entry point. Registers commands, detects Cursor via
│                       isCursor(), shows host-aware welcome notification.
│
├── VisualizerPanel.ts  Core class. Manages the singleton WebviewPanel,
│                       calls the Claude API, injects the bridge script,
│                       handles editor decoration sync.
│
└── promptBuilder.ts    Single exported function buildPrompt(code, hint).
                        Most important file — visualization quality lives here.
```

### How bidirectional sync works

1. `promptBuilder.ts` instructs Claude to call `window.__algoVizNotify(lineNumber)` after every `render()` call.
2. `VisualizerPanel._injectBridge()` appends a `<script>` before `</body>` in every generated HTML page. That script acquires the VS Code API and wires `window.__algoVizNotify` to `vscode.postMessage({ type: 'algoVizFrame', line })`.
3. `VisualizerPanel` listens for those messages and applies a `TextEditorDecorationType` to the matching line.

### Cursor detection

`extension.ts` exports `isCursor()` which checks `vscode.env.appName.toLowerCase().includes('cursor')`. Currently used only to tailor the welcome message. If you add Cursor-specific behavior (e.g. integrating with Cursor's chat or composer features), gate it behind this check.

## Build & run

```bash
npm install              # install devDependencies (TypeScript, @types/vscode, vsce, ovsx)
npm run compile          # tsc → out/
npm run watch            # tsc in watch mode (use alongside F5)
npm run package          # vsce package → algo-visualizer-cursor-0.1.0.vsix
npm run publish:openvsx  # ovsx publish (requires OVSX_PAT env var)
```

Open the folder in Cursor and press **F5** to launch the Extension Development Host.

## Settings

| Key | Default | Purpose |
|-----|---------|---------|
| `algoViz.apiKey` | `""` | Anthropic API key — required |
| `algoViz.model` | `claude-sonnet-4-6` | Model for generation |
| `algoViz.syncEditorCursor` | `true` | Enable/disable line decoration sync |
| `algoViz.autoVisualize` | `true` | Generate immediately on panel open |

## Key invariants to preserve

- **`promptBuilder.ts` is the source of truth for generation quality.** When improving output, edit the prompt — do not hard-code HTML in TypeScript.
- **Keep `promptBuilder.ts` in sync across `cursor-extension/` and `vscode-extension/`.** They should produce identical visualizations. If they ever diverge, document why.
- **One panel at a time.** `VisualizerPanel.currentPanel` is a static singleton.
- **No external bundler.** Plain `tsc` output (`commonjs`, `ES2020`) — no webpack/esbuild.
- **Bridge script must survive HTML variations.** `_injectBridge` inserts before `</body>` if present, else appends.

## Common tasks

- **Improve visualization quality** → edit `src/promptBuilder.ts`. Mirror the change in `vscode-extension/`.
- **Add Cursor-only feature** → gate it behind `if (isCursor()) { ... }` in `extension.ts`.
- **Add a new trigger** → update `package.json` (`contributes.commands` + `contributes.menus`) and register in `extension.ts`.
- **Change decoration style** → edit `createTextEditorDecorationType` in `VisualizerPanel` constructor.
- **Support other languages** → add a `language` param to `buildPrompt` and adjust the code fence + syntax guidance.

## Publishing

Cursor uses [Open VSX](https://open-vsx.org/) (not VS Code Marketplace). To publish:

1. Create an Open VSX account, generate a Personal Access Token at https://open-vsx.org/user-settings/tokens.
2. `export OVSX_PAT=<token>` (or `setx OVSX_PAT <token>` on Windows).
3. First time only: `ovsx create-namespace aerrami -p $OVSX_PAT`.
4. `npm run publish:openvsx`.
